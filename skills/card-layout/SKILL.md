# card-layout · 图文卡片排版出图器

> 把**已经写好的文字(+可选图片)**,排成小红书 / 公众号的 **3:4 图片连发**(1080×1440)。
> 定位:**纯排版出图的下游 skill**。不写文字——文字由上游写作 skill 或用户提供。

---

## 0. 三条铁律

1. **不生成文字**。接文字、排文字、出图。文字不够好,请用户回上游改,别自己瞎写正文。
2. **分阶段确认**。每个 Stage 给用户看、等点头,再下一步。绝不一口气从文字直接渲成图。
3. **智能在「读稿提炼 + 切卡规划 + 看图配图」**。价值不是套模板,是把文字提炼成节奏、判断每段最适合什么模块、把图配到对的段落,最后按选定风格渲染。

---

## 1. 总流程(Stage 0→6)

```
Stage 0  接稿        用户贴文字 +(可选)丢一堆图片 +(可选)账号配置
Stage 1  读稿提炼 ★  核心观点 / 读者钩子 / 页面节奏(见 §2)
Stage 2  切卡规划 ★  逐段判模块 + AI 自动把图配到对应段 → 编号规划表 → 用户确认/微调(见 §3)
Stage 3  选风格 ★    读 styles/index.json 按内容筛 → 展示 2-3 套 sample 预览 → 看图选(见 §4)
Stage 4  账号配置    品牌名 / handle / 栏目名;不提供则留空(见 §5)
Stage 5  渲染        读选中风格的 design.md,按模块逐卡出 1080×1440 PNG(见 §6)
Stage 6  自审+归档   拼图自检(溢出/对齐)→ 修 → 问存哪 → 归档(见 §7)
```

---

## 2. Stage 1 · 读稿提炼

通读全文,提炼并**先给用户看**:
1. **核心观点**:一句话压成最硬的一句(≤25 字)。
2. **读者钩子**:封面那句要勾人(方形封面压到 **4–14 字**,保留最强动词)。
3. **页面节奏**:拆成不重复的小节,排连发顺序。**字数定图量**:600–1000 字→5–7 张;1000–1800 字→7–9 张。

---

## 3. Stage 2 · 切卡规划(含自动配图)

### 3A 逐段判模块
基于节奏,逐段判定**模块类型**(每个模块各风格都有实现):

| 内容长这样 | 模块 |
|---|---|
| 一句最勾人的话(开篇) | **封面** |
| 一段叙述 / 长文段落 | **正文** |
| 一个词 + 它的定义 | **概念** |
| 一个数字 / 占比 | **数据** |
| A vs B / 以前 vs 现在 | **对比** |
| 第一步…第二步…(有先后) | **步骤** |
| 一句最想被记住的话 | **金句** |
| 三五个并列条目 | **清单** |
| 配图说明某段(图文互释) | **图文**(见 §3C) |
| 收尾 + 关注引导 | **封底** |

### 3B ★纯长文模式
若稿子就是大段叙述、没有明显封面/金句/数据 → **只用 `正文卡`**(必要时自动加 1 封面 + 1 封底),**不硬塞概念/金句/对比**。有啥模块用啥。
**字数上限提示**:小红书连发最多 18 张图,一张正文卡舒服承载 ~80–120 字 → **纯长文建议 ≤ 约 1500 字**;超了提示用户删减或拆成上下两篇。**不做每卡硬字数,靠切卡 + 渲染后自检兜底。**

### 3C ★自动配图(用户零标注)
图片和文字是**互相说明**的强关联,但**不要求用户标注**:
- 用户只管**贴稿 + 把图片一股脑丢进来**(任意顺序、不用标记);
- **我逐张看图内容 + 读全文,按内容相关性自动提议配对**(咖啡图→讲咖啡那段;文件名如 `morning.jpg` 也作线索);
- 在规划表里摆出提议:`03 · 图文(上图下文) · [咖啡图] · 配「早晨那段」`;用户扫一眼确认/改两处;
- 只在我实在判断不了时,才反问「这几张分别想配哪段」;
- **可选快捷**:若用户稿子本就用 Markdown `![说明](文件名)` 行内写好,就地沿用(可选,非必需)。
- **布局自动**:横图(16:9/1:1)→上下;1 竖图→左右;2 竖图→并排拼;3-4 图→网格(每页最多 4 张)。

### 3D 产出「编号规划表」
给用户过目,**建议用图/数据/对比处显式标注 + 理由**,确认后才往下。

---

## 4. Stage 3 · 选风格(看图选)

1. 读 `styles/index.json`(只读这个,别批量读 design.md);
2. 按内容**筛**:`fit_modules` / `fit_topics` / `density` / `mood` 对上的留下;
3. **展示入围 2-3 套的 `sample.png` 给用户看图选**(show, don't tell);
4. 用户选定后,**才读该套的 `styles/<slug>/design.md`** 作为渲染配方(渐进式披露)。

**10 套内置风格**(详见 index.json):
warm-editorial 暖衬线编辑 · swiss-grid 瑞士网格 · vintage-collage 版画长文 · folder-doodle 彩色文件夹 · bauhaus 包豪斯 · neo-brutalism 粗野 · minimal-quiet 性冷淡 · bold-poster 得意黑海报 · memphis 孟菲斯 · typewriter-memo 打字机便签。
每套都有:封面/正文/概念/数据/对比/步骤/金句/清单/封底(`page-*.html`)+ 4 图文版式(`page-img-ab/lr/duo/grid.html`)+ `sample-*.png`。

---

## 5. Stage 4 · 账号配置(留空规则)

开头一次性问、套用全卡:**品牌名**(顶部小标签)+ **handle**(页脚)+ 可选**栏目名**。
- **不提供 → 留空**(不渲染那个标签,而不是放假名);
- 一套里每张卡的页脚 + 署名**必须完全一致**(位置/对齐/格式/署名/颜色),照搬该套既有页脚。

---

## 6. Stage 5 · 渲染管线(踩过坑,照做)

每张卡 = 一个独立 `.html` → 无头 Chrome 截图成 PNG。铁律:
1. **别让 Chrome 直接往 iCloud 路径(`Mobile Documents`,带空格/中文)写 PNG**——会静默失败。先在 `/tmp/cardlayout_work` 渲染,再 `cp` 回归档目录。
2. **整段写在一次 Bash 调用**,字面量路径(跨次 Bash 变量不保留);该调用用 `dangerouslyDisableSandbox:true`。
3. 系统无 `timeout` 命令 → 后台跑 + 轮询 png 存在 + kill。
4. `--force-device-scale-factor=2` 出 2160×2880 高清;`--virtual-time-budget=13000` 等中文 webfont 加载完(否则回退系统字)。
5. 中文 webfont 走 ZeoSeven CDN(京华老宋 309 / 文津宋 376 / 得意黑 92)+ Google Fonts(思源/Playfair/Inter)。
6. 复用脚本见 `scripts/render.sh <html> <out.png>` 和 `scripts/shoot.sh <styledir>`(渲一套 + PIL 拼图自审)。

**视觉规格**:1080×1440;字体见各 design.md;照片 `object-fit:cover` + 圆角/框型随风格;插画 CocoMaterial 改色(见 `kit/`)。

> ⚠️ **排版铁律**(各 design.md 已固化,新建卡照守):
> ① 大标题加负 `margin-left` 抵消衬线/显示字左 side bearing(京华老宋 -0.06em / 文津宋 -0.09em / 得意黑 -0.05em;思源黑可忽略;居中布局不加)。
> ② 标题不加引号(中英文 ""/「」都不加)。
> ③ 标题 2 行内别挤 3 行。
> ④ 强底纹风格(neo 圆点等)正文/页脚垫实色块保证可读。
> ⑤ 图文卡要带本套招牌结构(typewriter MEMO 抬头+横线/vintage 纸纹+衬线题注),别退化成"大图+小标"。

---

## 7. Stage 6 · 自审 + 归档

- 渲完用 `scripts/shoot.sh` 拼一张总图,**自审**:溢出?对齐?页脚一致?有问题改 HTML 重渲。
- 回存前**必问用户存到哪个文件夹**(给建议默认让其确认):`AI-CoWork/Work/Content/卡片/{YYYY-MM-DD}-{关键词}/`,内含 `cover.png`+`01..0N.png`(连发顺序)+ 各 `.html`(便于重渲/改色)。

---

## 7.5 平台尺寸(小红书 / Instagram)

- **小红书 / 通用:`3:4 = 1080×1440`(默认)**。
- **Instagram carousel:`4:5 = 1080×1350`(推荐——占屏最高、互动最好)**;`1:1 = 1080×1080`(跨平台/投放备选)。
- **一条连发的所有卡必须同一比例**(IG 按首图比例裁剪后续图,混比例会被裁/留白)。
- **切换尺寸**:渲染时改 `--window-size=宽,高`,并把 HTML 的 `html,body{height}` 设成目标高度。模板按 3:4 设计:居中类卡(`top:50%`)自动适配;顶/底用 `top`/`bottom` 锚定的元素会自动贴边重排。4:5 比 3:4 矮 90px、1:1 矮 360px——渲前用 `scripts/shoot.sh` 自审一次,防溢出。
- 开头就问用户目标平台/比例,整套统一渲。

## 7.6 语言(中文 / 英文 / 双语)

- 本 skill **同样支持英文 / 双语文案**:每套风格都内置拉丁字体(Playfair / Inter / DM Sans / Archivo / Cormorant / JetBrains Mono / 得意黑拉丁等),英文直接渲染即可。
- 英文内容时:① 标题仍**不加装饰引号**(规则不变);② 中文专属规则(全角标点、盘古空格、`「」`)不适用;③ 长英文标题可自然 ragged 断行;④ 字号阶梯/层级规则不变。
- 中英混排:沿用各 `design.md` 的 `拉丁字体, CJK字体` 字体栈——英文走拉丁、中文落 CJK,自动切换。

## 8. 资产清单 + 扩展

```
card-layout/
├─ SKILL.md                 本文件(工作流)
├─ styles/                  ★风格库
│   ├─ index.json           风格目录(Stage 3 先读这个)
│   └─ <slug>/              每套一个:meta.json + design.md(完整配方)
│                            + page-*.html(各模块)+ page-img-*.html(图文)+ sample-*.png
├─ kit/  palettes.css / tone.css   (旧配色库,仍可用)
├─ assets/  illustrations(CocoMaterial 改色) / photos(占位照片)
└─ scripts/  render.sh / shoot.sh   (渲染 + 拼图自审)
```

**加新风格** = 复制一个 `styles/<新slug>/` 文件夹,改 `meta.json`(名字/适配题材/mood/密度)+ `design.md`(母题/配色/字体/招牌动作/页面集/do-dont)+ 渲出 `sample-*.png`,在 `index.json` 登记一条。**母题靠发参考图 → 7 点拆解**得到(配色/字体/招牌动作/氛围/版式/适合/复刻度)。
